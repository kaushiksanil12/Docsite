Trash & Recovery

  • Deleted files and folders are moved to Trash, not permanently deleted
  • Images referenced in deleted files are also moved to trash to keep your storage clean
  • Click Trash in the sidebar to view/restore/permanently delete items
  • Use Empty Trash to permanently remove all trashed items

Git Repo Auto-Sync

Docsite can automatically backup all your documentation to a private GitHub repository.

  • Click Sync at the bottom of the sidebar to open the setup panel.
  • Create a new empty repository on GitHub and paste the URL.
  • Personal Access Token (PAT): Provide a token from GitHub Settings to allow syncing from any machine without needing to log in to Git locally.
  • Every time you create, edit, or delete a file, changes are safely pushed to GitHub.
  • The green dot on the sync button means everything is up-to-date!

How to generate your GitHub PAT?

  1. Go to GitHub Settings → Developer Settings.
  2. Select Personal access tokens → Tokens (classic).
  3. Click Generate new token (classic).
  4. Give it a name (e.g., "Docsite-Sync").
  5. Select the 'repo' scope (Full control of private repositories).
  6. Scroll down, click Generate token, and copy it immediately!

Tip: This token acts like a password for this specific repo. It allows Docsite to sync your data even on new machines or VMs.
